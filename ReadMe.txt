導入方法
	■1ファイル単位の導入
		・導入したい VBA ライブラリ（bas clsファイル）をダウンロードする。
		・Microsoft Excel を起動する。
		・「Alt＋F11」 を押下して「Visual Basic Editor」を起動する。
		・「Ctrl＋M」 を押下して「ファイルのインポート」ダイアログを起動する。
		・ダウンロードしたファイルを選択する。

	■複数ファイルの一括導入
		・導入したい VBA ライブラリ（bas clsファイル）をダウンロードする。
		・あわせて VBA ライブラリ「Mod_ExcelModule」をダウンロードする。
		・Microsoft Excel を起動する。
		・「Alt＋F11」 を押下して「Visual Basic Editor」を起動する。
		・「Ctrl＋M」 を押下して「ファイルのインポート」ダイアログを起動する。
		・ファイル「Mod_ExcelModule」を選択する。
		・インポートされたモジュール「Mod_ExcelModule」を「Visual Basic Editor」側でダブルクリックでコードを表示する。
		・コード上で「F5」を押下して「マクロ」ダイアログを起動する。
		・マクロリストから「ModuleImport」を選択して「実行」ボタンを押下する。
		※この時点でエラーが発生し、実行できないケースがあります。「■実行エラー…の対策」を実施してください。
		・おめでとう！！！複数選択が可能なダイアログが起動します。
		・VBA ライブラリ（bas clsファイル）を複数選択して「開く」を押下する。
	
	■実行エラー「プログラミングによる Visual Basic プロジェクトへのアクセスは信頼性に欠けます」の対策
		～Excel2003～

		メニューから「ツール－マクロ」を選択し、「セキュリティ」をクリック。
		「セキュリティ」ダイアログボックスで「信頼できる発行元」タブをクリック。
		「VisualBasicプロジェクトへのアクセスを信頼する」チェックボックスをオン。
		「OK」ボタンをクリック。

		～Excel2007～

		Officeボタンをクリックし、「Excelのオプション」をクリック。
		「セキュリティ センター」をクリック。
		「セキュリティ センターの設定」をクリック。
		「マクロの設定」をクリック。
		「VBA プロジェクトオブジェクトモデルへのアクセスを信頼する」をチェック。
		「OK」ボタンをクリックして、「Excel のオプション」を閉じる。
		
		～Excel2010～

		メニューから「ファイル」を選択し、「オプション」をクリック。
		「セキュリティ センター」をクリック。
		「セキュリティ センターの設定」をクリック。
		「マクロの設定」をクリック。
		「VBA プロジェクトオブジェクトモデルへのアクセスを信頼する」をチェック。
		「OK」ボタンをクリックして、「Excel のオプション」を閉じる。
